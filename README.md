# Emile Fortier

I build small, inspectable developer tools.

My current work is focused on Rust-native tooling, local-first agent runtimes,
command policy, approval flows, journaling, and proof-oriented systems. I care
about tools that are understandable enough to debug, constrain, and trust.

## Current Direction

- Building a personal coding-agent runtime in Rust, with emphasis on local
  control, explicit approvals, bounded command execution, and readable session
  records.
- Exploring authorship and provenance tools for code: how writing activity,
  timestamps, signatures, and verification can make software work easier to
  reason about later.
- Keeping systems small until real workflows justify more surface area.

## Tools And Languages

Rust, Python, TypeScript, Git, Linux/WSL2, CLI tools, structured documentation,
and test-driven iteration.

## What I Value

- Clear boundaries over hidden power.
- Direct implementations over wrapper layers when the runtime matters.
- Evidence, logs, and verification over vague claims.
- Small systems that can be read, tested, and replaced when they are wrong.

Keel, my main current project, is being prepared for a public release. I will
publish it when the repo is cleaned up enough to represent the work honestly.
